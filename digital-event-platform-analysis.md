# Amici — Digital Event Platform Feasibility Analysis

*March 14, 2026*

## Core strategic premise

Agencies sell human hours. Amici sells software and network effects. Every event an agency runs requires proportionally more staff time. Every event Amici facilitates runs on the same platform with near-zero marginal cost — because the coordination burden sits with the customer or with a coordinator supplier, never with Amici.

---

## How the model works

### Supplier matching flow
Customer describes event → Suppliers respond to Amici with offers → Amici packages and presents the full event proposal to the customer. The customer sees one coherent event proposal, not scattered individual bids.

### Coordination responsibility
Amici never coordinates. The responsibility sits with either the customer (who self-coordinates) or a professional event coordinator (booked as an add-on through Amici). From each supplier's perspective, their point of contact is always the customer.

### Framing
Amici is a supplier network and matching platform that makes simple events fully self-service and complex events manageable.

---

## Three service tiers

### Tier 1 — Single supplier
Example: Chef for a company dinner. Fully self-service. Supplier contacts customer directly for any questions. No coordination needed.
**Amici's role:** Matching + offer presentation + payment.

### Tier 2 — Multi-supplier, low dependency
Example: Chef + photographer at customer's own office. Suppliers each contact the customer independently. Platform shares basic event details to all booked suppliers.
**Amici's role:** Matching + offer presentation + payment + shared event brief.

### Tier 3 — Multi-supplier, high dependency
Example: Chef + venue + transport + equipment. Platform recommends an event coordinator add-on. If customer accepts: coordinator becomes their logistics point of contact. If customer declines: they self-coordinate, and Amici doesn't guarantee a coherent event experience.
**Amici's role:** Matching + offer presentation + payment + smart coordinator recommendation + optional coordinator add-on.

**Key detail:** Event coordinators are not a marketplace. They are professionals Amici collaborates with explicitly. This is a curated relationship.

**Guarantee model:** When a coordinator is booked, Amici can guarantee a coherent event experience. When the customer declines the coordinator, Amici does not guarantee coherence; each supplier delivers their part independently.

---

## Smart coordinator recommendation rules

To be defined later. The platform should have a smart rule set that determines when to recommend a coordinator based on event complexity signals (supplier count, dependency level, venue type, guest count, event type, etc.). This will be designed as the product develops.

---

## What can be digitized

- Supplier matching and discovery
- Offer packaging and presentation to customers
- Standardized communication (event briefs, booking confirmations, payment)
- Timeline and logistics templates for common event types
- Reviews and quality signals (ratings, verified track records)
- Automated reminders and nudges for deadlines and logistics
- Shared event briefs visible to all booked suppliers

## What requires human involvement (but not from Amici)

- Cross-supplier dependency management → Customer or coordinator
- Real-time problem solving on event day → Customer or coordinator
- Subjective judgment calls (style matching, vibe) → Customer's own taste

## What Amici will need to handle manually (small ops footprint)

- Dispute resolution (reactive, not proactive)
- Supplier vetting at onboarding
- Edge cases and unusual requests (route to coordinator add-on or decline)

---

## Key investor message

Traditional event agencies can't scale because every event requires proportionally more human time. Amici's platform economics are fundamentally different: the matching, offer presentation, and payment infrastructure serves every event at near-zero marginal cost. The only human coordination that exists is either the customer's own effort or a professional event coordinator Amici collaborates with, never Amici headcount. This means Amici can grow event volume without growing team size proportionally.

### Competitive positioning
Traditional agencies are the main competitor. The framing: agencies sell human hours, Amici sells software and network effects. This is not about being cheaper than agencies. It's about having fundamentally different (and better) unit economics that allow Amici to serve a market agencies can't reach profitably.
